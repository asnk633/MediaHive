# Phase 10.5 Copy Rules & Lexicon

**Purpose:**
A human-enforced (and optionally tool-enforced) list of forbidden phrases and their neutral replacements. Policy guidance must read as environmental context, not as directives from an authority.

---

## 🚫 Forbidden Phrases
*These words imply authority, urgency, correctness, or danger. They must not appear in any conflict or sync-related UI.*

- **Instructive / Authoritative**
  - "Must"
  - "Should"
  - "Required"
  - "Recommended"
  - "Best option"
  - "Preferred"
- **Urgency / Threat-based**
  - "Violation"
  - "Warning"
  - "Error"
  - "Risk"
  - "Action Needed"
  - "Fix Now"
- **Comparative / Biased**
  - "Safer"
  - "Correct"
  - "Invalid"

---

## ✅ Allowed Phrases & Neutral Replacements
*Replace forbidden semantic framing with these informational alternatives.*

| Instead of... | Use... |
| :--- | :--- |
| "You **must** choose the server version" | "The server version **is currently linked to** the active campaign" |
| "It is **recommended** you keep local changes" | "Keeping local changes **would result in** overwriting the current draft" |
| "**Warning:** This violates Org Policy" | "This remote value **is associated with** the current Org layout" |
| "Please **fix** this conflict" | "Please **review** these versions" |
| "The **safer** option is..." | "This choice **may affect** downstream teams" |
| "Validation **Error**" | "Version **Difference**" |

---

## The Footnote Rule
All policy guidance copy should feel like a helpful footnote. If the sentence feels like it belongs in an alert banner or a warning dialog, the phrasing is wrong.
