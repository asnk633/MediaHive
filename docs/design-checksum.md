# 🔒 MEDIAHIVE DESIGN CHECKSUM (NON-NEGOTIABLE)

## Purpose
This checksum exists to prevent aesthetic drift, narrative UI, cinematic concepts, and “AI imagination.”
MediaHive is a professional SaaS workstation, not a concept demo.

Any UI change must satisfy ALL checks below.

---

### 1️⃣ CATEGORY CHECK (FAIL = IMMEDIATE REJECTION)
The UI MUST look like:
- A modern SaaS dashboard
- A tool people use 8+ hours a day
- Something comparable to: **Cofeed, Linear, Stripe Dashboard, Notion (admin views)**

The UI MUST NOT resemble:
- Sci-fi command centers
- Cinematic consoles
- Game HUDs
- Movie control rooms
- Dribbble “dark concept” shots

❌ **If someone says “this feels cinematic / futuristic / dramatic” → FAIL**

---

### 2️⃣ LANGUAGE CHECK (ZERO TOLERANCE)
#### Allowed vocabulary
- Dashboard
- Tasks
- Due Soon
- Pending
- Completed
- Overview
- Activity
- Requests
- Media
- Events

#### Forbidden vocabulary (AUTO-FAIL)
- Mission
- Vector
- Protocol
- Baseline
- Command Center
- System Status (unless literal uptime)
- Operational / Strategic metaphors
- Poetic phrases

**Rule:** If a school admin or media staff wouldn’t naturally say the phrase, it cannot appear in UI.

---

### 3️⃣ VISUAL NOISE CHECK
❌ **Forbidden (ever)**
- Film grain / noise overlays
- Fog / haze
- Background textures
- Vignette effects
- Atmospheric blur layers
- “Mood” backgrounds

✅ **Allowed**
- Flat or subtly graded surfaces
- Clean card elevation
- Minimal shadow for hierarchy only

**Rule:** If the background draws attention to itself → FAIL

---

### 4️⃣ DARK THEME CHECKSUM
#### Base palette (desktop)
- **Canvas:** very dark grey / charcoal (not pure black)
- **Panels:** slightly lighter than canvas
- **Cards:** slightly lighter than panels

#### Accent usage
- One primary accent color
- Used only for:
    - Active state
    - Primary actions
    - Important indicators

❌ **Forbidden**
- Multicolor accents
- Rainbow gradients
- Decorative glows
- Accent colors on non-interactive surfaces

**Rule:** Color is for function, not decoration.

---

### 5️⃣ DEPTH MODEL (STRICT)
Only 3 layers are allowed:
1. **Canvas** – page background
2. **Panel** – containers / sections
3. **Card** – actionable units

No extra pseudo-layers.

#### Elevation
- Shadow = subtle, directional
- No glow
- No halo
- No aura

**Rule:** Depth must be understandable in one glance, not felt emotionally.

---

### 6️⃣ TYPOGRAPHY CHECK
#### Required
- Modern sans-serif
- Neutral
- High readability
- Clear hierarchy (size + weight)

#### Forbidden
- Serif fonts
- Cinematic fonts
- Wide-tracked uppercase headlines everywhere
- “Poster typography”

**Rule:** If the font looks like it belongs in a movie title → FAIL

---

### 7️⃣ DASHBOARD STRUCTURE CHECK
The Home/Dashboard page MUST include:
- Clear overview widgets
- Due Soon / Pending list
- Empty state that is calm and neutral
- Predictable grid (rows & columns)

❌ **Forbidden**
- Symbolic tiles
- Metaphorical indicators
- Fake system metrics
- Decorative status cards

**Rule:** Every widget must answer: “What should I do next?”

---

### 8️⃣ RESPONSIVE SCOPE RULE
- Desktop / Tablet FIRST
- Mobile must remain untouched until desktop is approved
- No shared redesign across breakpoints

**Rule:** One surface at a time. No parallel redesigns.

---

### 9️⃣ APPROVAL GATE (MANDATORY)
Before implementation:
1. Provide static screenshots only
2. Side-by-side with the reference (Cofeed-style)
3. Explicit approval required

❌ **No:** Silent refactors, “Trust me” changes, or surprise builds.

---

### 🔐 FINAL CHECKSUM STATEMENT
If any design adds narrative, mood, symbolism, or cinematic flavor, it is **invalid**, regardless of effort or polish.
